月夜WordPress主题稍作修改

Posted on 2009-03-04 | In 网站相关

这两天 月夜 忙着新建另一个WordPress博客（为了防止点击党们的恶意破坏，月夜不会放出此博地址。谢谢朋友们的关注），为了省事，就选择了目前十分流行、功能十分强大的iNove主题。 照例，月夜在使用一款新的主题时总是会做些改动，即将主题本地化，iNove也不例外。当然，因为iNove主题比较强大，而且，因为是做专业博客，也没有太多时间专注主题，所以月夜并不想修改太多，因此改动的地方很少。但在改动过程中，还是有一些经验；而且因为目前使用iNove主题的博友很多，所以，月夜准备在接下来的文章中和大家分享一下iNove主题本地化的经验。但在此文，月夜只是简单叙述一下自己对现在使用的这款WordPress主题所作的一些小小改动。 1.修改了超链接的样式 月夜使用的这款菠菜主题，开始时超链接的样式和普通字符没有什么区别，使用起来很不方便；而且鼠标滑过时的颜色变成了暗红色，看起来也不太美观。后来，月夜进行了简单修正，统一修改成了超链接最常用的样式——使用了正蓝色。不过正蓝色的超链接在文章中看起来又太过显眼，破坏了整个页面的美观性。 这次看到iNove主题，发现里面的超链接样式看起来很不错。就修改成了它的样式，但为了醒目，月夜还是保留了鼠标划过时的正蓝色。 2.修改了related posts插件的显示样式 一直以来月夜都感觉博友们的相关文章插件用起来很爽，但一方面没有时间，另一方面月夜又天生有些惰性，所以一直以来也没有添加。 前几天，月夜刚好空闲，就顺势下载了插件，安装了上去。但无比郁闷的是，插件显示的相关文章看起来极不美观，让人觉得十分不爽。不过，月夜的水平有限，因为wp_related_posts插件只有一个PHP文件，我没有找到相应的控制显示样式的CSS文件。无奈，只有作罢，任其丑陋地显示在月夜的博客上。 刚好在iNove中看到了这个插件的显示效果，非常好看；在style.css文件中又找到了控制显示样式的代码，心想，原来可以这样做。于是，月夜就把iNove主题style.css文件中第949-993行的代码复制了出来：

/* related posts START / / when related posts with title / #related_posts { padding-bottom:10px; } #related_posts h3 { background:#E3E4E6 url(img/box.gif) 0 0 no-repeat; font-size:12px; padding:6px 12px; font-family:Verdana,”BitStream vera Sans”,Arial,Helvetica,Sans-serif; letter-spacing:0em; border-bottom:1px solid #CCC; } #related_posts ul { background:#F4F5F7 url(img/box.gif) 100% 100% no-repeat; padding:3px 12px 8px; } #related_posts ul li { list-style:none; padding:3px 5px; border-bottom:1px solid #E3E4E6; line-height:18px; } #related_posts .count { font-size:10px; } #related_posts .excerpt { color:#999; font-size:11px; display:block; } / related posts END */

将其放到月夜使用的菠菜主题style.css文件中相应的位置(注意需要根据自己的情况适当修改其中的url(img/box.gif)一句，以使图片路径正确)。保存之后，将iNove主题下img文件夹中box.gif文件上传到月夜的主题文件夹下的图片文件夹中，刷新页面，月夜终于告别了那可恶的相关文章样式，迎来了如今的美好面貌。

iNove主题本地化的几点经验(一)

Posted on 2009-03-04 | In 网站相关

如前文 月夜WordPress主题稍作修改 所述，月夜 最近在建立一个新的专业博客，而为了省事儿，在新的博客上使用了现在比较流行、功能又比较强大的iNove主题。尽管主题已经比较强大，但月夜还是按照习惯在使用前将其进行了简单的本地化。因为现在使用iNove主题的朋友比较多，月夜将自己的一点经验分享出来，希望能给后来者带一点方便。

在进行下面的叙述前，月夜先说明一下，我们使用的iNove主题版本是1.2.3。还需说明的一点是，在下面的修改中，所提及的文件以及文件信息均以未修改前的绝对位置为准，而不指修改中的相对位置。

1.拓宽整个页面

iNove主题本来的页面宽度只有936px，在当今主流分辨率均超过1280x1024的屏幕上显得比较浪费空间；而且文章页面的宽度太窄就没法插入相对较大的图片。因此，月夜就将主体稍微进行了一些拓展，扩展到了980px；与此相应，修改左侧文章页的宽度为649px（此处是为了给侧边栏的分界线留下空隙），侧边栏保持300px不变。

修改的主题文件夹下style.css文件的代码如下：

修改第151行为：width:980px;
修改第309行为：width:649px;

当然，与此相应，就要修改主题文件夹中img文件夹下的header_footer.jpeg文件的宽度，将其修改到与新的主体宽度对齐；而其中的light.gif，如果要求不高，则可以不做修改（当然，你也可以进行修改以使其更完美）。

2.修改文章字体字号

iNove主题中文章字体的字号实在太小，给网友访问阅读文章带来一些不便，所以下面我们将其字体稍微修改加大一些。

同样是修改style.css文件：

修改第24行为：font-size:14px;
修改第496行为：font-size:14px;
修改第611行为：font-size:14px;

但因为其中如首页、存档页等页面上文章标题下方的阅读评论、文章发布日期等的字号没有太大影响，所以没有修改。

3.修改博客题目和描述样式

iNove主题的博客名和博客描述显示效果让很多人感觉不是太满意，毕竟很少有人愿意去做像MG12的那种题目效果。月夜试着稍作修改：

修改第162行为：padding:15px 30px 21px;
从第172行开始添加：font-size:32px;font-weight:bolder;
从第164行开始添加：#caption a { color:#FFF; font-size:24px; }
将第185-192行的模块注释掉（或者删掉）;

接着，我们打开主题文件夹下templates文件夹下面的header.php文件，将其中的第5行修改为：

<?php bloginfo(‘description’); ?>

当然，你也可以适当修改如上的描述文字指向的页面（月夜指向的是关于页面）。

4.修改页面的title

很多人有修改新主题title以使得更改主题前后网页页面的title保持一致。iNove主题的title在templates文件夹下start.php文件的第42行，你可以根据自己的需要将其修改为自己想要的形式。月夜修改的代码如下：

<br><?php if (is_home() ) { ?><br><?php } else {?><br><?php wp_title(‘’); ?> | <?php } ?><br><?php bloginfo(‘name’); ?> - <?php bloginfo(‘description’); ?><br>

显示的效果是这样的：iNove主题本地化的几点经验 | 月夜 - 分享网络知识 · 享受快乐生活。

5.添加postviews插件

wp-postviews插件能够记录并显示你的博文被阅读的次数，以方便访客查看你最受欢迎的文章，很多朋友都在使用，当然，月夜也不例外。下面我们说说如何在iNove主题上添加postviews显示的阅读次数。

首先在style.css文件中第341行插入

.post .postviews,

接着在文件的第377行插入

.post .postviews {
background-position:0 -16px;
float:left;
}

在主题文件夹下的index.php文件第19行之后插入：

<?php if(function_exists(‘the_views’)) { the_views(); } ?>

与此相同，我们也需要在应该显示阅读次数的诸如存档页面，分类页面，标签页面等所对应的archive.php文件的相应位置添加上上面的代码。

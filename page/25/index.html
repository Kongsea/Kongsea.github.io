QQ影音V1.2版新鲜体验及若干建议

Posted on 2009-06-02 | In 时讯品评

腾讯公司QQ影音小组今日发布了QQ影音V1.2版本。考虑到在最近一段时间纷纷扬扬的DNS断网事件中被穷追猛打的暴风影音所处的窘境，此次影音小组发布的1.2版难免会让人以为是腾讯公司为了给暴风影音致命一击而仓促推出的版本。然而，尽管如此，此次推出的QQ影音最新版本还是给我们带来了很多惊喜，推出了不少让人引为亮点的功能。这些新增或改进的功能包括：

转手机/ MP4格式；视频压缩； 对字幕设置的一些修正； 对剧情连拍功能的一些修正； 对打开文件对话框的一些修正； 对播放列表的一些修正；等等。

下面请朋友们跟随 月夜 一起来一点点地详细试用体验一下QQ影音小组给我们带来的最新版本——QQ影音V1.2版本。

电子邮箱众生相

Posted on 2009-05-28 | In 时讯品评

早在数十年前，在IM和手机等通讯工具不甚普及的当时，电子邮件（E-mail）就已经不可或缺。数年前，面对IM迅猛发展的劲头，曾有不少自以为是的“先知先觉者”预测，E-mail会在很近的一段时间内惨遭淘汰，然而时至今日，在网络日渐发达并发挥重要作用的今天，尽管受到了各种其他通讯工具的强烈冲击，E-mail依然是每一位网民朋友不可或缺的必备品。 月夜 认为，就目前的形势来看，因其具有的先天优势，短时间内，E-mail仍然不可能会被淘汰。然而，尽管E-mail短期内不会出局，但我们认为，根据目前的态势分析，不需太久，就会有很多电子邮件服务商惨遭淘汰。 其实，电子邮件服务提供商被淘汰的事件早有发生，只不过对于这个淘汰目前没有一个明确的定义。月夜认为，如果一个电子邮件服务提供商提供的E-mail申请数量持续走低；其活跃用户数持续减少，邮箱知名度始终下降；其功能在相当长的一段时间内没有扩展，没有及时跟上其他E-mail服务提供商对用户十分必要且有益的创新功能扩展；则这样的电子邮件服务提供商基本上可以认为已被淘汰。 今天，月夜根据当前互联网上电子邮箱服务情况来为朋友们做一点评，细细盘点一下电子邮件服务提供商的众生相。

1.已被淘汰的电子邮件服务提供商

月夜记得，中国大陆用户使用的最早一批电子邮件服务提供商提供的E-mail，比如@eyou.com、@chinaren.com、@tom.com等等，尽管在其他方面可能仍有不错成绩，比如@eyou.com面向教育为高校用户提供的@edu邮箱，但应该基本上已经算是被淘汰出局。其实属于这类的E-mail还有很多，因其知名度较低，早已淡出公众眼眶，所以在此我们不再作详细介绍。 应该说，此类邮件服务提供商的出局很有借鉴意义，服务商中的后来者可以好好深思为什么他们会从一流的邮箱提供商走向穷途末路；为什么他们会被新生代的邮箱提供商淘汰出局；等等。

2.即将被淘汰的电子邮件服务提供商

这类邮箱提供商尽管目前可能如火如荼，有声有色，但就其在某些方面所表现出来的一些迹象，我们可以预料，他们可能会在不远的将来被淘汰出局。

1）网易邮箱

网易的免费邮箱产品包括我们常见的@163.com和@126.com等电子邮箱，就目前来看，因为网易是中国最大的电子邮箱服务提供商，似乎其前景一片光明。然而，可能是因为某些不足为外人道的原因，网易似乎有意放弃这个头衔。前一段时间，网易甚至冒天下之大不韪修改用户协议，使得网易可以在用户90天内不登录使用电子邮箱时，将邮箱服务和帐号收回。可就在此前，腾讯的邮箱服务刚刚将这个期限加长到了2年！

此外，在诸如邮箱无广告、邮箱功能扩展、听取用户意见等方面一直止步不前，使得用户对网易邮箱的忠诚度日益下降。

2）搜狐、新浪邮箱

这两个应该算是邮箱服务提供商中的难兄难弟，其邮箱服务从未被拿上台面，似乎也从未在邮箱服务上倾注过多精力和财力。不出意外的话，这两款邮箱服务将在网易邮箱之前被淘汰。

3.电子邮件服务提供商中的遗老

这一类别中的典型代表当推雅虎和微软。曾几何时，雅虎邮箱和微软的hotmail以及live邮箱是邮箱中名副其实的国王。雅虎邮箱庞大的用户数，hotmail邮箱高素质的拥趸，一直是其他邮箱服务商艳羡的对象。然而，时至今日，可能因为雅虎邮箱庞杂冗余的界面；可能因为微软邮箱致命的速度；而且此二者均未提供足够让用户满意的功能，这对电子邮箱服务提供商中的昔日王者正面临着一种比较尴尬的处境。 衰落的雅虎应该已无暇顾及邮件服务，我们似乎可以理解；可依然不知财大气粗的微软对此有什么打算，难道是不屑于更新和改进E-mail服务？

4.电子邮件服务提供商中的革命者

这头衔自然只能授予致力于电子邮件服务革新的Gmail。自从Google在2004年愚人节推出这一革命性的邮箱服务以来，Gmail早已深入人心，成了当前互联网上邮箱服务的代言人。推出时即提供高达1G的容量，至今依然时刻增长；各种创新型功能不断推出，Gmail总能给你意想不到的满足；高速的邮箱界面，可信赖的邮件收发服务；时至今日，Gmail依然是邮箱服务商中当之无愧的革命者和领跑者。

5.电子邮件服务提供商中的学习者和超越者

尽管腾讯的某些服务一直为人诟病，但腾讯推出的QQmail却是一个例外，除了某些丧心病狂的反腾讯分子之外，QQmail得到了几乎所有网友的一致认可。 虽然QQmail在某些方面的创新性相比Gmail略差，其中包括一些被某些网友疯狂追捧的华而不实的创新，QQmail已经在很多方面走在了Gmail的前面。比如绿色无广告，比如简洁方便的界面，比如邮件收发时的极度方便，比如贴心的贺卡服务，等等。 但是 月夜 觉得，QQmail相比Gmail最大的特点还在于其亲民，能够俯下身躯倾听所有用户和网民的意见以及建议。一个最贴切的例子是QQmail推出了一邮多名服务之后，听取网友意见而及时推出的Foxmail邮箱迁移助手。当然，其在近日推出的撤回邮件以及提醒功能等服务自然也应归于此列。 我们认为，得益于腾讯公司不计成本的投入，得益于腾讯邮箱团队的不屑努力，再加上广大网友的一致支持，QQmail在中国大陆区战胜Gmail只是时间问题。

6.电子邮件服务提供商中的隔岸观火者

不知道朋友们是否注意到了这样一个现象，尽管几乎所有的互联网公司都先后提供了邮箱服务，号称全球最大的中文搜索引擎的百度公司却始终未涉足此服务。月夜 觉得这可能是因为百度无法在邮箱市场发挥其先天优势，因此不便轻易涉足。 但请诸君试观百度目前已经提供的服务，在百度空间上，它肯定也没有足够把握能做成功，然而，它涉足了；在购物网站上，百度应该很难击败淘宝，也很难在已有淘宝和拍拍的市场上再分一杯羹，然而，它毫不犹豫地进入了，甚至不惜与淘宝翻脸；其他如SNS，百度估计也很难在诸侯林立的市场上做大做强，然而近期也传出了它欲涉足的消息；等等。从以上信息我们不难看出，百度似乎只对能直接获取较大收益的东西才教有兴趣；对于其他不能直接获益的东西，或者如近乎吃力不讨好的邮箱，尽管众多网友多次要求，百度也绝不轻易涉足。

邮箱提供商被淘汰出局的原因分析

其实，从以上对邮箱提供商众生相的分析中，其被淘汰出局的原因已经十分清晰，在此，月夜 就简单地总结罗列一下：

1.邮箱服务不够人性化

比如用户在较长时间内没有登录会被删除帐号；比如不（轻易）提供POP服务等等；

2.邮箱服务没有一个良好的意见反馈通道

用户是邮箱服务的最大资源，用户意见尤其应收到邮箱开发者的重视，如果用户在使用邮箱的过程中发现的问题无法反馈并得到解决；如果邮箱开发者不能从用户哪里获得建议，没有这样的良性发展过程，邮箱服务肯定不能尽如人意；

3.邮箱界面不够美观，载入或登录速度较慢

如果用户不能快速登录，及时收发邮件，其他的一切服务之于邮箱都是空谈；

4.邮箱没有新功能扩展或扩展较慢

比如如果哪款邮箱当前还不支持POP服务、IMAP服务，或者不支持https安全链接，则无疑这样的邮箱不会得到用户的支持，其用户忠诚度也将不断下降，该提供商被淘汰出局应该只是时间问题。

用好WordPress不可不知的函数（六）

Posted on 2009-05-15 | In 网站相关

距上一篇介绍WordPress函数的相关文章已经过去了二十多天，今天，月夜就再次为朋友们送上一篇介绍WordPress函数的文章，仍然属于月夜的WordPress函数文章系列。本文接上文从这一系列的第36个函数开始，主要介绍的是一些比较杂乱的内容，使用上依然主要用在WordPress主题中。包括获取WordPress博客细节的几个函数，以及用于构架WordPress主题大框架的几个函数。 36.wp_get_post_tags()

该函数用来在某个文章页面或者根据某篇文章的ID来获取该文章的tag，获取的结果被放置到一个tag数组中。一个常见的使用方式如下：

if (is_single()){ $keywords = “”; $tags = wp_get_post_tags($post->ID); foreach ($tags as $tag ) { $keywords = $keywords . $tag->name . “,”; } echo $keywords; } 首先判断是否是单文章页面，如果是，则据当前文章的ID（$post->ID）来获取当前文章的tag，然后取得其name（$tag->name），并将其组合输出。

37.single_cat_title()、single_tag_title()

如名所言，这两个函数用来获取分类页面和tag页面的title，其通常的使用方式如：

<?php $str = single_cat_title(); echo $str; ?> 和

<?php $str = single_tag_title(); echo $str; ?> 然而，除此之外，single_cat_title()还可以用来在tag页面上获取当前页面的title；但single_tag_title()却不可用于获取分类页面的title。

38.get_settings()、get_option()

此二函数与前文用好WordPress不可不知的函数（三）中函数20.get_bloginfo()类似，使用方法也相同，可以通过后跟各种参数来获取WordPress博客的相关信息。 比如如下的调用方式：

get_settings(‘name’)或get_option(‘name’)

可以用来获取当前WordPress博客的标题。

39.wp_head()

该函数与前文用好WordPress不可不知的函数（五）中函数32.wp_meta()相同，是WP主题为WP插件留下的API Hook。

40.get_header()、get_footer()、get_sidebar()和comments_template()

这几个函数是用来在WordPress主题中获取并包含相应的文件的。比如：

get_header()用来包含当前主题文件夹下的header.php；

get_footer()用来包含主题文件夹下的footer.php；

get_sidebar()用来包含主题文件夹下的sidebar.php；

comments_template()用来包含comments.php。

需要注意的一点是，如果当前主题文件夹下缺少对应的文件，则函数会使用wp-content/themes/default/文件夹下的对应文件代替。 此外，以上函数后面都不能跟参数，只有get_sidebar()例外，因为一个主题中可以使用多个sidebar。形如get_sidebar(‘up’)的调用方法可将sidebar-up.php侧边栏模板文件包含到主题中。 除了以上几个函数之外，在主题中如果想包含一个具体的文件，还可以使用如下方式：

include(TEMPLATEPATH . ‘/***.php’)

如上的函数形式可以将当前主题文件夹下名为***.php的文件包含进来；其中TEMPLATEPATH是当前主题文件夹地址的一个引用（不含末尾的/，所以需要添加上）。

41.have_posts()、the_post()

这两个函数的使用范围有限，通常在WordPress的循环中使用，用以获取所有文章。其固定使用形式如下：

<?php if (have_posts()) : ?> <?php while (have_posts()) : the_post(); ?> 此处显示文章 <?php endwhile; ?> <?php else : ?> 此处显示未找到文章时的信息，比如404相关 <?php endif; ?> 另一种常见的形式是将如上代码中的前两行组合起来（其他地方不变）：

<?php if (have_posts()) : while (have_posts()) : the_post(); ?>

该形式通常会在模板的index.php、archive.php或者single.php等页面使用。除此之外的其他地方，我们通常不会看到此二函数的身影。

月夜盘点2009年4月中国互联网事件

Posted on 2009-05-13 | In 时讯品评

4月的中国互联网依然热闹不减，《明朝那些事儿》结局，阿桑和“沙僧”去世，NBA 季后赛开打，百度先后创造性地推出老年搜索和凤巢系统，甲骨文即将收购 Sun，Windows 7 RC 版放出，所有这些，无疑又为我们送上了一个有声有色的独特四月。当然，其中最热闹非凡的莫过于月中魔兽世界代理权的更换，以及月末猪流感的突然来袭。 下面，就请朋友们跟随 月夜 一起来回顾一下这个绚丽多姿的4月里发生的那许多不可不说的事儿。 1.阿桑、“沙僧”逝世

4月6日，4月12日，著名歌手阿桑和西游记中沙僧的扮演者闫怀礼分别去世，在互联网上掀起一片纪念热潮。 阿桑是来自台湾的歌手，以代表作《寂寞在唱歌》和《叶子》等驰名。闫怀礼是大陆的老一代演员，尤以在电视剧西游记中扮演沙僧而出名。

2.百度推出老年搜索

4月7日，百度高调推出号称“30多名杰出工程师经过近半年的研发、测试”的作品——百度老年搜索。该产品试图为老年人提供更好的搜索和上网体验。但经网友试用发现，此产品只是大字号、无广告的百度搜索，最多也就是在页面上添加了一些导航链接。因此，产品推出后饱受质疑，但迄今百度官方未就此事发表任何看法。

3.《明朝那些事儿》结局

4月11日，一部从2006年在网上推出即受到疯狂关注，写作过程中得到万千网民关注的作品——《明朝那些事儿》，在经过作者历时3年的努力后，终于宣告终结。在全书的最后一节中，作者当年明月以徐霞客的经历告诉我们，历史属于每一个普普通通的人民，我们每个人都是历史的主角。

4.魔兽世界代理权更换

4月16日，美国暴雪公司正式宣布，将网游《魔兽世界》在中国大陆市场的代理权交给网易，为期3年。魔兽世界大陆市场代理权争夺大站宣告结束。在这场对事件各方都很重要的争夺战中，网易获得最终胜利，九城被推向绝境。但也有舆论认为，此次代理权争战，争夺双方不过是鹬蚌相争，最终受益的只是坐山观虎斗的暴雪公司。

5.博鳌亚洲论坛年会开幕

4月17日，博鳌亚洲论坛2009年年会在中国海南博鳌开幕。本届年会的主题是“经济危机与亚洲：挑战与展望”。与会者就如何应对当前国际金融危机下亚洲所面临的挑战，怎样寻求金融创新及等问题进行了探讨和交流；并对2009年经济形势发展趋势进行了展望。 本次会议期间，不时传出惊人言论，诸如著名影星成龙称“我们中国人是需要管的”；著名房地产商任志强称“和工资比房价等于没涨”等等，引起诸多质疑和热议。

6.NBA季后赛开打

北京时间4月19日，NBA季后赛开打。姚明迎来了职业生涯的第四次季后赛之旅。所幸，没有麦蒂的休斯敦人在姚明以及火箭众球员的努力下，最终在第一轮中战胜了开拓者队，12年后重回季后赛第二轮（5月1日）。

7.百度发布凤巢系统

4月20日，饱受争议的百度竞价排名正式升级为“凤巢系统”。百度希望通过这种方式将搜索结果中的推广信息与非推广信息区分开来，早日从网民对百度的信任危机中走出来。但据月夜观察，百度此举并没有取得太好的效果，推广信息与非推广信息依然混淆不清。百度竞价排名升级为“凤巢系统”不过是换汤不换药的改个名称而已。

8.海盗湾事件

4月17日，瑞典法庭在著名IT共享网站海盗湾侵权案中，判处海盗湾创始人等4名相关人员一年监禁，并处以360万美元罚金。此案的判决结果在全世界范围内引起广泛抗议，在互联网上掀起了不小的波澜。

9.几款操作系统更新

4月，微软新一代操作系统 Windows 7 的RC版放出；Linux操作系统中的代表 Ubuntu 也推出了最新的9.04版本。在操作系统的选择上，网民朋友们有了更多的选择。

10.猪流感爆发

4月底，一场罕见的猪流感在墨西哥爆发。截至月末，猪流感已在墨西哥和美国致死了上百名患者，并已经向全世界范围内广泛传播。这场疫情给灾难深重的世界经济形势带来了更加恶劣的影响。但目前疫情尚未波及中国大陆（5月13日补充最新消息：中国大陆今日又确诊了一例病例；至此，大陆已经有四川和山东两例猪流感确诊病例）。

用好WordPress不可不知的PHP知识

Posted on 2009-05-10 | In 网站相关

已经连续数天没有更新博客，至于知识类文章的缺失更是长达半月之久，可能是最近一段时间心情有些郁闷，但原因又说不出来。当然，人的心情总不能每天都高亢，但不因心情低落而影响其他工作才是一种比较好的习惯。于是，今天，月夜 就试着继续更新博客，权且作为送给天下所有母亲的礼物，祝所有的母亲都身体健康，幸福快乐！ 在今天的文章里，月夜试着为朋友们讲述一些在使用WordPress的过程中不可不知的PHP知识。当然，因为使用WordPress对PHP的要求并不高，所以这里讲述的其实是最基本的PHP知识；因此，如果你是PHP高手，或者在使用WordPress的过程中对PHP知识用起来已经不再困难，则你完全可以忽视本文。可是，如果你对PHP知识一窍不通，或者在使用WordPress的过程中已经为此掣肘，则月夜强烈推荐你认真阅读本文，并籍此获得一些PHP知识，使自己在使用WordPress的过程中能够更得心应手。 首先需要说明的一点是，本文假定朋友们具有了至少一种编程语言（最好是C语言或者C++语言）的基本知识，因为PHP语言和其他语言的语句、变量等基本相同。而如果你不具有这方面的基础，要想把PHP语言叙述到能够在使用WordPress的过程中自如应用，则不是如本文这样的一篇小文所可叙述清楚的了。 学习PHP知识，你应该确保自己有一个PHP的学习调试环境。在进行下面的讲述前，首先确认自己已经按照月夜前面的文章安装了本地环境。有了环境之后，下面我们就来简单了解一下在使用WordPress的过程中不可不知的PHP知识。 1.PHP语言的结构

PHP语言的代码可以直接嵌入到静态的HTML页面中，但嵌入到页面中的PHP代码有一个结构，需要放置到指定的标记中。在静态HTML页面中，PHP代码的标记是以<?php开头，并以?>结尾。 比如，在如下的HTML代码中：

<?php wp_title(‘ | ‘,true,’right’); ?> <?php bloginfo(‘name’); ?> - <?php bloginfo(‘description’); ?> <?php wp_title(‘ | ‘,true,’right’); ?>、<?php bloginfo(‘name’); ?>和<?php bloginfo(‘description’); ?>分别各为一段PHP代码。 此外，标记PHP代码还有其他方式，比如其中的一种形式以<?开头，以?>结尾，但通常情况下，我们还是应该（而且很多人在）使用默认的以<?php开头的方式。

2.PHP语言的语句

从如上1中的代码实例可以看出，每一条PHP代码的语句都以分号（;）结尾（尽管在只含有一个PHP语句的代码块中，PHP语句末尾的分号可以省略，但月夜绝不推荐！）。下面再举一个PHP代码例子：

<?php bloginfo(‘description’); ?>

如上的HTML代码中，包含了两个PHP代码块，从中可以看出，PHP代码和HTML语句完全可以无缝对接。

3.PHP的注释

尽管PHP提供了多种注释方式，但月夜只推荐朋友们记住其中的两种方法，而且，通常在实际应用中，人们只会用到这样两种方法。这两种方法和C++语言的注释类似，分别为单行注释和多行注释：

//此为单行注释，以//开头；

/*此为多行注释，

包含在/和/之间*/

4.PHP的变量

除了不需要显式声明之外，PHP语言中的变量形式也和其他常用语言不同，PHP的变量总是在变量名称之前添加一个$符号，至于变量名，则和其他语言的规则相同。下面是几个PHP变量的例子：

$title、$description、$counter

下面是一个应用了PHP变量的PHP代码块：

<?php if ($post->post_excerpt) { $description = $post->post_excerpt; } else { $description = mb_substr(strip_tags($post->post_content),0,120); } ?> 此外，还需要注意一点，PHP的变量区分大小写。

5.PHP语言的输出和字符串变量

这里我们将这样两个东西放到一起讲述，因为输出十分简单，只有在输出字符串变量时，才需要稍加注意。 PHP语言的输出语句通常使用的是这样两个：

print()和echo()

Yueye.org is a good site Yueye.org is a good site Yueye.org is a good site Yueye.org is a good site $title is a good site\n $title is a good site\n 在PHP语言中，字符串的连接（拼接）使用的是如上例子中的点号（.），它可以将两个字符串拼接成一个字符串。PHP语言中的输出语句，可以输出其后双引号（”）或单引号（’）中包含的内容，也可以输出未包含在引号内的变量内容。此处单引号和双引号的区别显然也可以看出：双引号内部的内容中，变量和转义符都会得到解析；而单引号内的内容，变量和转义符也会原样输出。

6.关于PHP语言的其他需要注意的地方

关于PHP语言，月夜就为你简单介绍到这里，其实，在使用WordPress的过程中，对PHP知识只需要简单了解一些即可，有如上的一些知识，即已足够。下面，我们再简单叙述一下PHP语言和其他语言（比如C语言、C++等）的不同之处。 1) 数组

PHP语言的数组有两种形式，一种是数值键（也就是通常的形式），另一种是关联键。

a.数值键：

形如$price = array( 0 => “1.8”, “1” => “2.5”, “2” => “3.2”);或者$price = array(“1.8”, “2.5”, “3.2”);的定义方式定义的是使用数值键的数组，因为PHP的数值索引也从位置0开始，所以上面两种定义方式效果相同。

$price[1]引用的数值都是2.5。

b.关联键：

形如$price = array( “pear” => “1.8”, “apple” => “2.5”, “peach” => “3.2”);的定义方式定义的是使用关联键的数组。

可以用形如$price[“pear”]的方式来引用”1.8”。

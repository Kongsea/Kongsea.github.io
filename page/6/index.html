近来那些烦恼事儿

Posted on 2013-08-27 | In 心情日记

很喜欢跑步的那种感觉，以至于当我在几年之后重拾跑步的时候，很轻松地就找到了那种愉悦，每日晚间，围操场跑行数十圈，心旷神怡，无以形容。于是买了跑鞋，装了手机软件，兴致勃勃地跑起来的时候，心里甚至想着要跑一个马拉松。 然而不过跑了数周，甚至刚刚找到以前跑步的那种感觉，膝盖却已经有了不适感。屈伸膝盖时能轻易听到膝盖骨骼间传出的脆生生的声响，我不知道这到底是怎么回事儿。网上搜索，发现这就是臭名昭著的跑步膝，困扰着很多跑者。而其原因，无非是跑前跑后没有热身，或是大腿小腿的肌肉不够发达以支撑运动起来的身体，故而导致很多重量都集中在了膝盖间而致膝盖受伤。 知乎上有一个问题，下面的答复相当好地把这个问题给描述来出来，而且还给出了很多解决方法。仔细阅读下来，发现自己除了要减少跑量（毕竟我在短短数周内已经将跑量从2公里加到了5公里，而且还常在周末加跑一倍；更何况我还依仗着自己的基础一周多能跑步五六次，而没有给身体以足够的休息），还要多多锻炼身体的肌肉，尤其是核心力量锻炼，比如plank，比如深蹲。 此外，我还按照有些人的经验买了护膝，跑步时穿上护膝，虽然感觉不到明显的好处，但听说略有效果。可是更多还听说人体会对护膝形成依赖，一旦养成，去掉护膝时，膝盖将更易受伤。对此，我倍感困惑。 更且，坚持两周下来，膝盖并没有太多改善，屈伸膝盖关节时，声响一如既往的发出，甚至手抚膝盖能感觉到膝盖关节处的很多骨头的来回错位移动，我感觉到一种空前的恐惧。8月里迄今我一共跑步3次，一共不过只有区区十公里，照此下去，不要说计划中的半马和明年的马拉松赛，我甚至不知道今生是不是将要与跑步绝缘。 跑步不力中，时间却已经来到了8月中旬，这已经是沈阳的夏秋之交，秋风肃杀，灭绝一切活性。而于我，这恐怖却在于，自从五六年前来到沈阳，每年此时我都会感冒不止，以至于持续一月之久或者更长，流鼻涕打喷嚏，嘴干舌燥，嘴内上颚疼痛不止，甚至吃饭都成了问题。而除此外，眼睛每每还干涩 这恐怖的一月，对我来说，每年都是梦魇，即令我的身体没有那么脆弱，即使我用尽全力去看去治疗，最终也绝不会有什么好转，而只能空待时间过去，病魔凋零。 还好，我只能将此归结于幸运，今年我了解了一些东西，在药店里的工作人员说我可能是鼻炎，因为没有感冒能够持续一月有余。而我在网上搜索发现这季节性鼻炎，或者称过敏性咳嗽，与我的症状极其相似。在感冒药没有任何效果的情况下，无奈中买来鼻炎喷雾稍试发现竟很快有好转。而干涩的眼睛时而痒的难受，经常遍布血丝，也只好弄些眼药水使用。 不过终于，我弄清了自己的病因，我知道了为什么每年此时都会频发这些症状，虽然暂时不能治愈，而且甚至我都拿它没招，但这一切还是终于慢慢地朝着好的方向在前进了。我们不会放弃自己，更不会就去投降，如果你真的去投降了，那你才是真正的失败！

LibSVM在MATLAB中使用时的几个问题

Posted on 2013-07-28 | In 大学相关

最近在科研中需要用到支持向量机（Support Vector Machines, SVM）来进行分类，而目前比较成熟的用于实现SVM的软件包则首推LibSVM。LibSVM目前的版本已经能直接在MATLAB中使用了，虽然MATLAB已经提供了诸如svmtrain等SVM相关的函数，可是相比LibSVM的易用性以及其强大的功能还是要差了不少。不仅如此，目前有关LibSVM的资料在互联网上也比较丰富，所以最终选择LibSVM在MATLAB环境下实现SVM分类样品会是一个很不错的选择。 LibSVM可以直接从其主页上下载到，我们所需要用到的就是MATLAB版本，不过仍然需要对其进行一个编译。这个过程十分简单，不过需要你已经安装了MATLAB的较新的版本（比如2009以后的版本，因为较旧的MATLAB版本不能识别较新的VS，而只能使用VC++ 6.0），以及Microsoft Visual Studio。

1.编译LibSVM的MATLAB版本

如果你的电脑上已经安装了如MATLAB的较新版本，以及Microsoft Visual Studio，那么你可以直接将下载的LibSVM解压出来并拷贝到MATLAB的工作目录下（这个过程不是必须，但会更为方便接下来的处理）。然后在MATLAB中将LibSVM目录添加到MATLAB的识别路径下（Add path），并将LibSVM下的MATLAB文件夹设置为MATLAB的当前工作目录。

接着在MATLAB命令行窗口中输入mex -setup，回车执行命令选择编译器。需要注意的是mex跟-setup之间有个空格。不要选择MATLAB的默认编译器，而手动选择Microsoft的VS编译器（只需告诉MATLAB其安装路径即可）。

设置好编译器，在MATLAB命令行窗口中输入make，即可开始对LibSVM进行编译了。如果没有什么错误提示，最终编译成功之后，在LibSVM文件夹下的MATLAB文件夹里将有四个以.mexw32结尾的编译完成的可执行文件。

2.加载数据集验证程序

编译完毕之后，我们可以加载数据集以验证程序是否编译成功。但是这里就出现了问题，如果使用以前的很多教程和文章中出现的诸如

load heart_scale

来载入heart_scale数据文件，将会出现如下所示的错误：

??? Error using ==> load Number of columns on line 2 of ASCII file …\MATLAB\libsvm\heart_scale must be the same as previous lines.

有人说好像这是因为heart_scale数据集是一个C++类型的文件，所以使用MATLAB的load载入而不能成功。

所以，现在需要使用LibSVM提供的libsvmread函数来读取数据文件，读取格式如下：

[label_vector, instance_matrix] = libsvmread(‘heart_scale’);

问题是，这样仍然不能成功，而提示如下的错误：

can’t open input file heart_scale

但是我们可以解决这个问题，通过这样的方式：

1.将位于LibSVM文件夹下的heart_scale数据文件拷贝到当前工作目录（MATLAB）中；

或

2.将LibSVM文件夹设置为当前工作目录；

如此，再执行libsvmread命令，即可将数据文件读取进来进行一些基本的处理了，使用

model = svmtrain(label_vector,instance_matrix);

训练SVM模型；

使用

[predict_label, accuracy, decision_values] = svmpredict(label_vector,instance_matrix,model);

得到模型的预测结果：

Accuracy = 86.6667% (234/270) (classification)

我要跑个马拉松

Posted on 2013-07-23 | In 心情日记

我在高中，就比较喜欢跑步，或许是因为我的性格，比较文弱，不善交际的我对于其他需要团队的运动似乎压根儿提不起精神来；而没有运动，身体在繁重的学习中何以能保持健壮，或者仅仅保持健康。那时的我也就是每天早上起来在早读前跑上三五圈，不过也就是一公里多的距离，只能算是一个简单的健身。 我在大学，学习上已经轻松了很多，虽然依然在努力着去学，可时间上毕竟富余了不少。因着高中的经验，我知道大学里的自己必须时刻注意着自己的身体。所以每晚从教室勤工俭学回来之后，我都会换上鞋子和衣服，下楼去宿舍旁边的土操场上跑步。那时的我穿的不过只是一双最普通的飞跃运动鞋，何曾想过保护自己的身体？但因为时间限制，每晚我只在操场上跑3圈，算是一公里稍多；不过我的速度却始终是自己的极限。 所以两年下来，我竟然以此方式积累了一个很棒的跑步身体，而在后来的院里、学校的各种运动会上获得了不错的成绩和很好的名次。然而后来就放下了，我至今仍然不知道为什么当时我竟然一下子放下了自己坚持了那么多年的好习惯。甚至直到后来我去到科大，来到所里，也只不过是偶尔间断性地跑上几天，早将大学里立下的诸如要跑个马拉松的宏志抛到九霄云外。 所幸，前一段时间我在潘石屹的微博里看到了他每每分享的使用Nike+记录的运动情况，我搜到了这款软件，以及其他的运动记录软件，这些软件记录你每一次的运动情况，还有诱人的运动轨迹。我知道这多少有一些炫耀心理，但看着自己一步步跑下来所形成的运动轨迹，那是一种说不出的美妙的感受。 而再幸，我又在网上看到了一些关于长跑的保护方面的文章，我知道了自己在大学里那种无知者无畏的锻炼方式有多么恐怖。如果你想长跑，切莫抱着一种“那就去跑吧”的心里匆匆上路，除非你想后半生病痛缠身。为什么要那样呢？我们跑步，无非是为了锻炼，为了健康，为了“生命在于运动”。所以，我们需要保护好自己不能受伤，我们需要一双好的跑鞋。有了一双适合自己而且能给予自己很好保护的跑鞋，那么，那就去跑吧！ 我对长跑，有一些基础，大学里参加的10000米环校跑还曾获得过不错的成绩。所以一旦跑起来，速度还算可以，而没有感觉到累的自己，总是设想能跑更远的距离。自然，马拉松进入了我的脑海。

每一个长跑者都有自己的马拉松梦，可沈阳却没有自己的马拉松比赛。但附近的营口有，丹东也有，不过很遗憾，今年的比赛都已经结束；而明年，应该是四五月间的事儿了。国内比较有名的马拉松赛有北京马拉松、厦门马拉松、上海马拉松和大连马拉松，下半年貌似还有一个北京马拉松，是10月20日的赛事。档次高，风景美，想一想从天安门前跑过该是多么豪迈的事情，所以北马是国内每一个马拉松爱好者的梦。而我，如果从梦开始，那又该是多么美的事儿。 但北马报名很难，花费也很高，而且仅有不到3个月的赛前训练时间，做起来会是比较匆忙，冲动中去做这样的事情，着实不太妥当；最重要的，今年我自己学业紧张，万不可投入过多的精力和时间在这个事儿上。 纠结中得知长春9月7日在净月潭公园有一个长春国际友好城市半程马拉松邀请赛，那么就轻松多了，不需要报名费，距离也只有20公里左右，去长春也不至像去北京那样不易。而且，净月潭的风景和空气质量都更为适宜跑步，5A级国家森林公园内完成自己的第一个半程马拉松该是多么幸运的事情，所以暗自庆幸的我毫不犹豫地报上了长春净月潭半程马拉松赛。 那么，今秋跑了这个半程马拉松，明春就可以在营口鲅鱼圈马拉松和丹东马拉松中选择而跑一个全马了。这样的计划，循序渐进，既能实现自己的愿望，也不至对生活有过多影响，可谓是两全其美的事儿了。

说说微信公众号的信息推送：何不把选择权交给用户？

Posted on 2013-07-17 | In 评测建议

最近一段时间微信5.0即将推出，而其传言更改微信公众号的信息推送机制，更是引起了许多微信公众号的经营者迫切关注。关于信息推送机制的改变的具体传闻如下：在最新版本微信5.0中，微信公众号将会被分为两类，订阅号和企业号，除了将分门别类地被放到两个文件夹中之外，订阅号跟目前一样，最多每天只能主动发送一条消息；而企业号则将更为尴尬，每月只有一条消息的发送权限。 传言出来，让很多微信公众号的持有者和经营者大呼无奈，也引起了很多人的反对。虽然如此，微信公众号对用户造成骚扰的情况将会被极大限制，但微信公众号还有多大存在的意义？ 不过，对用户来说，这个问题就十分简单，如此的信息推送机制，是否会影响到用户正常使用微信公众号？是否会让微信公众号对用户的价值变小？下面，月夜 就从一个普通的微信用户和微信公众号的关注者的角度来说说微信公众号的信息推送机制。

Excel中“您尝试打开的文件的格式与文件扩展名指定的格式不一致”的解决方法

Posted on 2013-07-03 | In 经验知识

最近在使用微软的最新版Microsoft Office 2010软件打开一个从网站上汇总下来的Excel表格时，出现了一些问题，对话框提示：

您尝试打开的文件“***.xls”的格式与文件扩展名指定的格式不一致。打开文件前请验证文件没有损坏且来源可信。是否立即打开该文件？

提示信息如下图所示：

一直在使用微软的Excel软件，但使用中基本上也没有出现过类似的提示。那么，我就试着点击对话框中的“是”，来打开文件。可随即就出现了错误：

Microsoft Excel提示：“文件已损坏，无法打开”。而我记得清清楚楚的，以前这种汇总的文件在Excel 2003中可以轻松打开，并非文件被损坏。显然应该是这个新版本的Excel 2010对文件的检测问题。 在网上搜索许久，都未能解决问题，因为很多方法解决的只是让上面的第一个对话框消失，而对这种报文件已损坏，无法打开的问题都没有解决。所幸我自己琢磨许久，终于最终解决了这个问题。

1.为文件取消受保护的视图

1.首先点击Excel 2010中的“文件”菜单项下的“选项”菜单，打开Excel选项对话框。 2.然后点击选项对话框中的“信任中心”选项，并在其下点击“信任中心设置”按钮，打开“信任中心”对话框，如下图所示：

3.接着在信任中心对话框中选择“受保护的视图”自选项，从中取消选择“为来自Internet的文件启用受保护的视图”。点击确定保存设置。

经过如此的设置之后，再次打开刚刚的Excel文件，同样会出现如上第一图中的提示，不过选择点击其中的“是”按钮，即可打开该Excel文件，正常进行接下来的操作。

2.取消文件格式与文件扩展名是否匹配的检查

上面第一个图中的提示是由于Excel中开启了文件格式与文件扩展名格式之间匹配的检查，为了去掉那个提示，关闭那个对话框，在以后打开类似文件时，那个对话框不再出现，我们可以这样操作：

1.回到Windows的桌面上，单击开始->运行->输入regedit->确定，打开注册表编辑器，找到注册表子项 HKEY_CURRENT_USER\Software\Microsoft\Office\14.0\Excel\Security； 2.点击这个Security，然后在注册表编辑器窗口右侧的空白处点击右键，从中选择新建-DWORD(32-位)值，将其取名为ExtensionHardening； 3.右击刚刚新建的ExtensionHardening注册表项，然后选择修改，将注册表值修改为0，点击确定，关闭注册表。如此，取消扩展名与文件格式之间匹配的检查。

然后打开刚刚的Excel文件，可以看到，既没有第一个图中的提示，也没有文件损坏的提示，而能直接打开文件进行正常的编辑了。

几款运动记录软件简评

Posted on 2013-06-25 | In 评测建议

夏天到了，天气越来越热了，而朋友们似乎对健身的需求越发迫切了。或傍晚，或周末，公园里，小河边，走路或跑步的人们络绎不绝，有人为着健康，有人为着体型，他们都在努力地坚持运动着。 然而说起运动，就有坚持，可大多数人面对这项很需要毅力的东西时，都会难以坚持下去；再或者运动没有记录，像没头苍蝇一样误打误撞，三天打鱼两天晒网。还好，伴随着智能机的普及，这些曾经困扰着我们每一个人的问题，现在有了很好的解决方法。 目前在智能机上比较流行的一些运动记录软件，或记录运动本身，或描述运动轨迹，或分享运动给朋友们以资监督，已经在很大程度上让我们的半途而废有了很大改观。这些软件已经有了很多，国外比较著名的有Nike+、Runtastic、runkeeper、endomondo等等；因为这些国外软件对于中国国情的不了解，或者了解较慢，国内的追随模仿者也有了不少空间，比如咕咚运动+、跑步控等等。 今天，我们就来看看这些软件的功能如何，其在使用中会碰到什么问题，而最关键的，到底这些软件中哪一款最适合自己。我们从运动记录软件中的功能开始逐项点评：

1.运动记录

如名所言，对一款运动记录软件来说，最重要的功能当然首推记录运动的功能。这些功能包括记录每一次运动概况，记录每一次运动轨迹，等等。

1.1 运动轨迹

运动轨迹是你运动中的运动轨迹在地图上的直观反映，比如下图：

可以看到，这样的运动轨迹记录会让自己在运动过后特别有成就感，我们不能不说，这种轨迹是这类运动记录软件的最大创新。但是在目前基本上所有运动记录软件都有运动轨迹记录功能时，有些软件竟然反人类地没有这一功能，比如跑步控，那么，我们只能抛弃它了。

1.2 运动轨迹的准确性

运动轨迹独具一格，显然，我们希望运动轨迹能准确反映我们的实际运动，虽然偶有偏差，但最好不要太大。可另一些软件，比如Nike+，上面1.1中引用的就是Nike+记录的运动轨迹图，显然有些问题，我们在公园里的徒步走，竟然给记录到穿越青年大街在楼宇间的绕圈走了，这偏差，实在是让人有些无语。 不过仔细查看发现，或许这是因为Nike+引用了Google地图，而Google地图在中国的普通地图和卫星图之间有一些距离上的误差。我们在卫星图上查看记录的运动轨迹：

可以看到记录相当准确，的确显示我们在公园里绕圈走了一段距离。 问题是，虽然Nike+记录的轨迹在卫星图上显示较为准确，可通常直接呈献给用户的却是普通地图，所以，用户对Nike+的第一印象就会是，记录不准确。 因此，针对这种情况，我们建议Nike+尽快修复这一问题，要么使用其他地图，要么修复这一问题。 而同期我们试用的其他几款软件，如Runtastic，runkeeper等，均记录了较为准确的轨迹，比如这款国内的软件——咕咚运动+，这是其记录的运动轨迹：

1.3 轨迹记录的持续性

可是准确还不足够，我们还需要轨迹记录一以贯之，不要丢失，不要没有持续性。比如，这是咕咚运动+记录的某一次运动轨迹：

1.4 运动轨迹的美观性

上面我们附了几个图片，分别是Nike+和咕咚运动记录的运动轨迹，但是显然，Nike+记录的轨迹更美观大方，也更平滑好看，而咕咚运动的轨迹准则准矣，可毛刺毕现，似乎不够美观典雅。

2.记录分享

现在是一个社交网络统治的时代，没有什么东西不能被分享到社交网络上，而且不应该有什么东西不能被分享到社交网络上。而对一款运动记录软件来说，分享到社交网络上尤其必要，我们不只可以养成一个好的习惯，更可以让社交网络上的朋友监督自己。

2.1 分享到社交网络

遗憾的是，在分享到社交网络这一功能上，国外的几款优秀软件几乎全军覆没，要么它们不支持分享到社交网络上，要么只能分享到国外通用的Facebook和Twitter上，而对国内比较流行的社交网络，比如微信、微博，几乎没有支持。 Nike+本地化较好，但目前也不过只支持分享到新浪和腾讯微博上；咕咚运动+可以分享到包括微信在内的其他应用上，但分享内容单调乏味，而且分享形式毫无人性化可言，比如分享到微信上的竟然是一个链接，点击打开一个记录的页面。我想问一下，需要这样推广吗？分享一张记录运动轨迹和运动摘要的图片，再由用户搭配些文字不会更好？况且，感兴趣的，不需要推广便会主动询问；不感兴趣的，如此分享会被别人当作垃圾，谈何打开页面详细查看。

2.2 灵活分享

我目前还没有在这些运动记录软件中看到诸如截屏分享的功能，想一下吧，有时候我们需要自定义，我们需要更灵活，为什么不加一个截屏的功能呢？比如我自己调到运动轨迹记录页面，然后我截下这个轨迹图，我便可以自如地在微博、微信上分享，这是我的社交网络，我需要符合我自己习惯的风格。

3.其他问题

3.1 消耗能量

运动中消耗了多少热量，我想这是减肥一族很关心的问题，可遗憾的是，我们发现目前这些应用对消耗热量的计算十分白痴。如果我选择一项运动，它不会计算我的速度，不会计算我的运动强度，便直接用运动形式跟运动时间相乘便得到我的消耗热量。你能想象吗？这些应用竟然用这种形式去糊弄那些热衷减肥的人群，可怜他们还虔诚地说着，我又消耗了多少大卡！

3.2 更好的历史数据呈现

我想要这样的Android桌面应用软件

Posted on 2013-06-13 | In 谈天说地

我使用的Android手机是摩托罗拉的defy+，一直以来因为Android自身系统所带的桌面效果和可定制性的问题，我都没有使用Android系统自身的桌面。但囿于defy+有限的内存（RAM），我的手机也没有自如选择一款Android桌面应用软件的权利，毕竟稳定性和平滑性才是这类软件最重要的东西。 最开始使用的是名气很大而且为人熟知的GO桌面，但你知道，GO桌面由桌面软件而不断推送推广其他软件的行为让你很头疼；再而且，当时的GO桌面没有桌面小工具，我无法灵活方便地开关网络、GPS等等。 所以后来换成了腾讯桌面，但这款软件更为垃圾，不过详情我已经不记得，但其用起来竟然不如GO桌面。不过还好，这时候我碰到了QQ桌面Pro。灵活的可定制性，方便的桌面小工具，再加上其一款雨后清晨的桌面主题，让我自第一次试用起，便再没有更换过桌面应用软件。还需要特别说明的是，QQ桌面Pro后来把名字改成了QQ桌面。

问题由来

然而，Android系统上应用软件的通性就是，开发者总想以一款软件之名行所有软件之实。远如我们都能知道的墨迹天气，近到能占用上百M内存的正点闹钟。QQ桌面没能免俗，在最新的2.9版本中，竟然加入了清理系统内存的功能。而且，这清理不能自己定制，它会在你操作过某软件之后，自作主张把驻留在内存中的进程终结清理掉。

当然，有时候我们可能觉得这一功能不错，但问题就出在你作为一款桌面应用软件，干嘛要管用户的内存。而最重要的是，它没有提供任何定制性。这样，就会出现这样的情况，比如有时候我用相机360拍完照片之后，退出了相机软件，但它会保留一个进程在内存里写照片到ROM上，可是QQ桌面通常会自作主张终结这一进程，导致我们拍摄的照片未能保存。 面对这样的QQ桌面，我们只能更换，可是在月夜搜寻其他桌面应用软件的过程中，却始终未能找到一款称心如意的应用，那么，究竟什么样的应用才是我们想要的Android桌面应用软件呢？

我想要这样的Android桌面

1.平滑稳定

作为一款驻留内存的软件，桌面应用必然不能占用太多内存，不然系统的平滑流畅就无法得到保障。而稳定性自然也是一款桌面软件的基本素养。

2.提供丰富的桌面小工具

在墨迹天气以及各种天气应用软件日渐庞杂时，用户通常不会考虑特别安装一款天气软件，而这时候，提供一个桌面天气插件就是一款Android桌面的必须了。此外，诸如灵活关闭打开网络、GPS、WiFi等的小工具自然也很必要。 不过，像QQ桌面这次升级中所提供的清理内存，我想考虑到几乎所有Android用户都会安装如LBE等手机管理软件，则QQ桌面越俎代庖似乎没有必要。

3.可以灵活自如地添加移除应用

在有些桌面软件中往Android桌面上添加应用时，竟然需要一个一个地添加，而竟然不能多选一次添加数个，这种反人类的设计如何赢得用户？而批量移除当前桌面上已经添加的应用的功能似乎在Android桌面上暂时还没有。

4.循环往复滑动桌面

月夜的Android桌面上一共有三屏应用，主屏在中间，左右各一屏，这样可以保证在任何情况下，最多只需要滑动一次即可来到目标应用所在的屏上。然而有些Android桌面软件竟然从屏的最左侧无法再往左滑动而进入到最右侧一屏，这样的设计简直让人无法理解。

5.锁定桌面

手机被别人使用时，尤其是孩子们拿起来玩儿，有时候很容易就把整个手机桌面上的各种应用搞得混乱，为什么不添加一个锁定手机桌面的功能呢，当桌面被锁定时，桌面上的应用无法被删除或挪动，而用户想进行类似操作时，可以首先进行解锁。

6.安分守己

首先用户使用一款手机桌面软件，只是为了取代系统丑陋而定制性不足的桌面，所以任何一款Android桌面应用都应该首先认清自己的定位，在桌面定制性方面多做功课，而不是越俎代庖，添加庞杂的功能，比如管理手机、比如管理手机上安装的软件等等。因此，作为一款Android手机桌面应用软件，安分守己是最基本的底限，也是用户最起码的要求。

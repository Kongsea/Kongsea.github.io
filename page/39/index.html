上传完成之后，登录HastyFree空间的控制面板，我们来查看空间的数据库信息。登录地址形如：http://51blog.hastyfree.com:2082。登录之后，创建MySQL数据库以及数据库用户名，并记录下相应的信息，比如51Blog的数据库信息如下：

然后，我们来修改上传到服务器中的根目录下的config.inc.php文件，打开后找到文件的14到17行，如下：

$dbhost = ‘localhost’; // 数据库服务器 $dbuser = ‘username’; // 数据库用户名 $dbpw = ‘’; // 数据库密码 $dbname = ‘discuz’; // 数据库名

$adminemail = ‘admin@your.com’; // 系统管理员 Email

迄今最好的拼音输入法

Posted on 2008-12-08 | In 经验知识

时至今日，网络以及计算机软硬件都得到了空前的发展，对我们一个普通的网友来说，当然也获得了丰厚的便利。今天，51Blog就来介绍一下对我们普通网友来说所必不可少的拼音输入法。 虽然五笔已经诞生多年，而且也得到了相对广泛的应用，然而对我们一个普通的非专业人士来说，拼音输入法依然是我们输入汉字的首选。我们肯定还记得当年使用智能ABC输入法的情形，51Blog觉得记忆最深的可能就是输入个常用的汉字却要翻数页候选字，而且每次翻页还要低下头去寻找键盘上的+、-键，比如这个“祝”字；我想诸位肯定还记得当你换台电脑，却只能接受冰冷的不符合自己习惯的输入法的经历。诸如此类的情形肯定让你头疼不已，当然，也自然会痛恨不已。 然而，大概三四年前，尤其是时至今日，这一切彻底地改变了。随着一个个优秀的拼音输入法的诞生，我们终于有了更多的选择，紫光、拼音加加、搜狗、Google以及QQ都先后涉足拼音输入法领域，为我们带来了种种优异的输入法。可是，这些输入法在带来方便的同时，也给我们带来了一个让人头痛的问题：究竟应该选择哪款输入法？ 其实，正如那个经典的80-20理论所言，这种选择最艰难的地方也正在这最后的20。尽管拼音输入法层出不穷，然而真正能吸引眼球，让人喜爱的不过也就聊聊数种，而且在这仅有的几种拼音输入法中，也可以很轻松地排除一些。比如，没有网络备份功能的拼音加加，功能有限创新不足的紫光等等。反正在经过51Blog的一番努力之后，拼音输入法的候选本上只剩下了搜狗、Google以及QQ三种。 其实51Blog开始时使用的是拼音加加，后来搜狗出现时，并没有更换，主要是个人比较讨厌这个名字；但Google拼音一经出现，我就立马放弃了原来的选择，毕竟拒绝新事物的行为是要付出代价的。使用Google拼音大概一年有余，开始时用着还是挺不错的，而且觉得像Google这样的大公司，后续研发力度肯定不小，也肯定会持续发布功能更强，更利于我们使用的版本。正如拼音输入法深度评测一文中所言，当时Google还有一个智能选词的优势，觉得使用起来要比搜狗和QQ方便顺手一些。 但让人遗憾的事情发生了，从去年开始，Google在相当长的一段时间内没有发布任何新的版本，使得Google拼音像个没娘的孩子一样，让人几乎对其失去了信心。而同期，搜狗和QQ拼音频繁更新，频繁加入新的功能，频繁进行不懈的研发。尤其是搜狗拼音，本来就和Google几乎不相上下甚或稍胜一筹，再加上专注的精神以及不懈的努力，更使得我对Google拼音丧失了信心。其实即使这些，仍未让我产生放弃Google拼音的想法，毕竟更换输入法并不是一件小事，而且多年养成的习惯也不那么容易舍不得丢失。最终的致命一击是今年九、十月间我使用Google拼音网络备份时动辄会出现问题，比如本地词库的首词经常在备份之后被改变，给我造成了极大的不便。 于是，今年十月左右，我终于痛下决定，将Google拼音输入法从我的生活中清除。开始时我和很多朋友一样，也是安装了搜狗输入法，然而，遗憾的是搜狗和Google的词库不能互通，这样我就必须丢弃我的Google拼音输入的习惯，重新在搜狗中建立。无奈，我只有再去试用QQ，十分幸运，QQ能导入Google的词库，于是，我可以方便地将Google的习惯导入到QQ中。就这样，我开始了QQ拼音的历程。 还算可以，经过两个多月的努力，现在用QQ输入法已经十分顺手。写作此文时输入法的速度也稳定在90字每分钟左右，使用输入法已经没有了什么障碍。而且原来一直为我所诟病的不支持英文输入的问题，也在QQ拼音输入法 v1.5 发布时得到了解决，现在已经能十分方便的在中文输入情况下输入英文单词了。而且还有着不错的记忆功能，一旦你输入一个系统没有的单词，下次重新输入时，就已经被加到了词库中。不过还是有些问题，比如QQ中文输入法和英文输入法没有切换的快捷键，只能使用鼠标去切换，有点不方便。但我相信，QQ输入法肯定也会慢慢更新、慢慢提高的。让我们共同期待吧。可是，最近QQ拼音输入法的更新速度有了点问题，好长时间了，就更新了一次，发布了V1.5版，不过考虑到该版本所作的改变，还是值得支持的。 然后再来说说Google拼音，不知道是什么原因，自从我十月抛弃Google拼音之后，Google拼音在短时间内进行了罕见的两次大的更新：

WordPress安装完成后需要做的几件事（三）

Posted on 2008-12-04 | In 网站相关

在前面的文章中，51Blog已经带朋友们初步了解了WordPress安装完成之后要做的一些事情，比如WordPress的基本设置、为WordPress更换主题、安装WordPress插件，等等。本文中，51Blog将继续为大家讲述WordPress安装完成之后需要做的其他一些后续工作。

一、设置WordPress博文的分类

这个设置事实上十分简单，当然，也不需要我来多言。然而，51Blog还是要和大家分享一下分类的两条经验：

1.添加分类时为分类写一个英文分类缩略名

事实上这点并不是十分必要，但主要是为了得到一个友好的链接，毕竟一个这样的分类链接：https://www.yueye.org/category/freesource总比这样的链接要友好一些：https://www.yueye.org/category/%e6%9b%b4%e6%96%b0（该链接不可访问，请朋友们不要点击^_^）。 同样，当你为WordPress博客添加一个标签时，最好事先添加好，并且为其取好英文缩略名，这样你就可以得到一个友好的链接地址了，比如51Blog网络硬盘分类的链接：https://www.yueye.org/tag/levodisk。

2.删除WordPress博客文章分类中的未分类

下面51Blog为你简单介绍一下如何删除WordPress博客默认的文章分类：“未分类”，这是WordPress安装完成之后系统默认的文章分类。但这个分类没有太大意义，而且还可能有点碍眼，所以，下面我们就来将其删除。但直接点击“管理”-“分类”，在分类中并无法删除，因为在未分类的前面没有复选框，无法选中将其删除，如下图所示：

这主要因为“未分类”是WordPress博客的默认分类，默认的东西，自然没法删除。但我们可以首先来修改WordPress博文的默认分类，将其设置为非默认，然后就可以在“未分类”前面看到供删除选中的复选框了。需要注意的是，在接下来的操作之前，你必须保证已经创建了新的分类。创建好新的分类以后，我们点击“设置”主菜单下的“撰写”菜单，如下图所示：

如上图，我们可以在默认文章分类右侧的下拉列表中选择默认的文章分类，只要选中的不是未分类，然后点击页面下方的保存修改按钮，即完成了默认文章分类的设置。 此后，回到刚刚“管理”主菜单下的“分类”子菜单中，如下图所示：

如上图，我们即可选中未分类前面的复选框，然后点击删除按钮来删除这个讨厌的未分类了。

二、修改删除WordPress友情链接中的默认分类Blogroll

和上面的文章分类中的“未分类”一样，这个友情链接分类Blogroll也没有太大的意义，所以我们可以将其删除。但同样因为它是友情链接中的默认分类，我们无法直接将其删除。所以我们还是需要首先修改一下WordPress友情链接的默认分类。当然，在进行下面的操作之前，我们还是需要先创建一个新的友情链接分类，而且51Blog建议你最好为新建的友情链接分类取一个英文缩略名。比如，分类名称可以取为“博客联盟”，英文分类缩略名可以取为“blogunion”等等。新建好相应的友链分类之后，我们还是转到如上面所述的“设置”主菜单下的“撰写”菜单，如下图：

如上图，我们将默认链接分类修改为除Blogroll之外的其他分类，比如我们的友情链接，然后点击页面下方的保存修改，即成功修改了WordPress默认的链接分类。 然后，我们回到“管理”菜单下的“友情链接”子菜单，如下图所示：

如上图，我们即可以直接选中Blogroll，然后点击删除按钮将其从我们的WordPress博客中删除了。当然，删除了这个分类之后，属于该分类的友链会被转移到默认的友情链接中，所以我们还需要点击上图中的链接菜单，去删除自己不需要或者不想要的其他链接，比如WordPress自带的那些链接。删除成功之后，我们就完成了对WordPress友链分类的设置了。

原来使用的是echo，将HTML代码一个个输出，看来我的知识实在是太匮乏了啊。不过正如原文中所言，对于属于多个分类的文章来说，会显示多张图标，用着不是太好，于是，我就直接在末尾加了一句break，显示一张图标之后就直接退出。又适当修改了一下主题的CSS代码，最后得到的总体效果还算是可以。 可正如原作者在Wordpress 显示分类图片的补充一文中所言，如上的代码还略有不足，主要是鼠标悬停于图标上时，显示的文字不是我们设置的汉字分类，而是我们设置的分类的英文缩写。原作者将上面的代码修改如下：

<?php foreach((get_the_category()) as $cat)
{
$catname =$cat->category_nicename;
$cattitle=$cat->cat_name;
echo “<a href=/category/“;
echo $catname;
echo “/>”;
echo “<img src=http://yourdomain/wp-content/cat-icons/";
echo $catname;
echo “.jpg alt=\“$catname category \“ title=\“$cattitle\“ />\n”;
} ?>

这样，使用上面的代码，并在最后一个?>之前添加上break;语句，即可完美为文章添加分类图标。最终实现的效果可以参看51Blog。

WordPress新手问答（三）

Posted on 2009-04-16 | In 网站相关

距上一次的新手问答又有相当长的一段时间了，因为这一系列的文章推出后反响不错，很得朋友们的支持，所以，今天，月夜继续为朋友们送上这一专题的第三篇文章。当然，月夜期待能继续得到朋友们的支持，也希望朋友们能多为月夜提供这一专题的素材和资料。月夜先谢谢朋友们。 今天这个专题里，我们仍然围绕月夜博客上的一些设置进行讲解和说明，以方便朋友们了解月夜的设置，方便自己的订制。 七、问：月夜，我在你博客上留言评论时为什么不能像你或者其他人那样拥有如下图这样的头像呢？

需要怎么进行设置才能拥有自己的个性头像呢？具体应该怎么设置呢？ 答：这项在WordPress博客上提供头像的服务统一由一个网站提供，据月夜推测，该网站可能与WordPress有合作关系。 具体设置如下，首先打开服务提供网站：http://en.gravatar.com。 在该网站上注册一个帐号，记得在注册时填写的E-mail一定要是你在WordPress博客上评论时经常使用的那个E-mail帐号。该网站提供的服务通过E-mail帐号来识别帐号，显示头像。 注册好帐号之后，即可将自己的头像上传到网站上，然后进行适当设置，以后在所有的WordPress博客上留言评论时，只要你填写了相应的E-mail帐号，即可在博客上显示自己的个性头像。不过在你刚刚创建帐号，上传头像之后，其生效可能会有一段时间的延时。具体的效果可以参看上面的图像，也可以参看月夜博客上朋友们评论时显示的头像效果。 八、问：我博客首页上的文章总是显示全文，显得很庞杂，为什么不能像月夜博客一样只显示文章的一部分内容呢？ 答：在博客首页上显示摘要，点开具体的文章之后显示文章全文，这样设置可是使整个博客布局比较美观简洁。 要想实现在博客首页上显示摘要的效果，通常可以有这样两种方法： 1.修改主题模板文件

修改主题模板文件夹中的index.php文件，将其中的

<?php the_content(‘’); ?>

修改为

<?php if(!is_single()) { the_excerpt(); } else { the_content(__(‘(more…)’)); } ?> 或者

<?php the_content(); ?>

不过与此对应，你还需要修改主题文件夹下其他文件中的对应内容，比如archive.php或者archives.php等等。然而使用这种方法我们无法灵活控制摘要中显示的内容，因此月夜没有使用；而且，因为修改不易，也不推荐朋友们使用这种方法。

2.使用<!–more–>标签 在需要分割的地方插入<!–more–>标签，这样，就可以将标签前面的内容设为摘要，只在首页显示；标签后面的内容在文章页中可以正常显示。 添加方法有以下两种：

1) 手动添加代码：在WordPress后台编辑文章时，切换到HTML代码界面，在其中需要添加标签的地方直接插入<!–more–>标签即可； 2) 点击图标插入：WordPress后台的文章编辑框上方的按钮中有一个插入<!–more–>标签按钮，如下图：

将鼠标放置到正在编辑的文章需要放置标签的地方之后，直接点击该按钮，即可在文章中当前鼠标所在的位置上添加<!–more–>标签。

答：设置成如上图所示的效果十分简单，不过分为两种情况： 1.使用归类或文章存档widget（小工具）的主题：

通过WordPress后台的“外观”-“小工具”菜单进入到小工具管理页面，找到右侧的“分类目录”小工具，点击其右侧的“编辑”链接，打开设置页面，如下图：

在如上图所示的设置页面中，选中显示文章计数前的复选框，即可在分类目录后显示属于当前分类的文章计数，实现如上上图月夜博客的效果。日期归档的设置与此类似，月夜在此不再赘言。

2.未使用widget，而使用wp_list_cats和（或）wp_get_archives的主题：

使用非记事本编辑器（最好用Notepad2）打开主题文件夹下的sidebar.php文件，找到其中的如上两个函数，将其中的参数添加上&optioncount=1或show_post_count=1。 月夜设置的参数具体如下：

<?php wp_list_cats(‘sort_column=name&optioncount=1&hierarchical=0’); ?>

<?php wp_get_archives(‘show_post_count=1&type=monthly&limit=8’); ?>

如此，即可在归档项目后面显示相应的计数。关于如上二函数中的其他参数，月夜将在稍后为朋友们继续介绍。

月夜推出WP评论关键词替换插件

Posted on 2009-04-15 | In 网站相关

月夜博客上的评论显示规则一直以来都是这样设置的，如果同一位作者评论一次以后，在下一次评论时，即可不经审核直接显示；但如果其中含有月夜设置的关键词的话，则无论何时均需审核。 可是不知道到底是什么原因，除了偶尔有一两条不经审核直接显示的评论之外，绝大多数评论都需要等待月夜审核之后才能显示。月夜考虑，可能是因为审核关键词设置的太多，也可能是因为评论的朋友无法被WordPress识别为同一个人。不管怎么样，如此的评论显示方式，很是打消朋友们的评论积极性，而且审核评论过程中，还会给月夜带来很多烦恼。 尤其是最近，月夜博客每天都有很多热心朋友进行留言和评论，当然，月夜非常感激朋友们的支持。但因之而来的就是评论审核的繁重任务。于是，月夜思考，是否有这样的一款插件，可以将评论中的特定关键词（也就是敏感词汇）替换为指定的字符。我在Google上搜索了很久，没有得到这样的结果。只有一款插件，但插件中将所有的关键词统一屏蔽为了相同的字符，这显然会有很大的不便。 于是，月夜就试着自己写了一个插件，命名为WP-Comments-Filter。该插件十分简单，而且因为月夜的水平有限，目前也十分简陋，没有很好的易用性。主要作用是将评论中出现的某些关键词（这些关键词可以预先设置）替换为对应的字符。这些替换规则可以自己预先定义，而且可以灵活的添加删除。 需要注意的是，使用该插件后，尽管在前台的访客看到的是替换后的字符字样，但在WordPress后台，我们可以看到留言或评论的本来面貌。而且，在你更改替换规则之后，以前已经替换的字符还会自动重新变化为修改规则后的字符。 插件比较简单，目前功能也比较单一，而且尚未经过完整的长期测试。因此，月夜目前不准备将其放出。如果经月夜自己使用效果比较好的话，等过一段时间，月夜会将其充实发布出来。 最后，谢谢朋友们对月夜的支持，谢谢你们的留言和评论。当然，也欢迎朋友们留言测试。

月夜盘点2009年3月中国互联网事件

Posted on 2009-04-14 | In 时讯品评

3月里的中国也不清静，“两会”激情上演，“3.15”揭出的动人心魄的黑幕，朴树迷离的“香水女孩”事件，Windows 7操作系统一泄再泄掀起的风潮，所有这些，都给我们的互联网生活带来了多姿多彩的体验。现在，就请朋友们跟随 月夜 一起来回顾总结一下2009年3月的中国互联网事件。 1.“两会”召开

照例，每年3月，两会都会如火如荼的上演。今年3月3日和3月5日，全国政协和全国人大为人熟知的“两会”正式拉开帷幕。虽然明着看说什么关注民生，关注改革，关注金融危机，关注教育公平，等等；其实，两会也就那么回事儿，不过是走走过场而已。然而，所谓“三个女人一台戏”，更何况那么多的委员们呢？于是，每年一次的大戏就映入了我们的眼帘。有牛人如钟南山，鞠躬尽瘁针锋相对的针砭时弊，也有刘翔等人的照例缺席；有恬不知耻的“中国根本没有发生金融危机”的言论，也有真正为民请命者的声音；更有如要求“春运火车票提前半年出售”般跳梁小丑一样的话语。3月13日，随着人大会议的谢幕，这场大戏宣告结束。

2.嫦娥一号撞月

北京时间3月1日16时13分10秒，我国第一颗探月卫星——嫦娥一号在地面人员的控制下，成功撞击月球，为中国探月一期工程画上了句号。据悉，嫦娥一号卫星于2007年10月24日由西昌卫星发射中心发射升空，11月7日准确进入了距月面200公里的环月工作轨道，正式开始执行探月任务。从发射升空到如今撞月，嫦娥一号探月卫星已经在太空飞行长达494天，超额完成了预期任务。

3.“3.15”晚会揭黑

尽管政府公信力逐年下降，人民对政府的信任一日不如一日，但每年有政府喉舌央视推出的“3.15”晚会无疑还是会受到人们的关注。其原因可能归于这样几点，一是不管是真干还是忽悠，每次晚会总会为人民做点事情；二是这种晚会可以看作是鹬蚌相争，而作为旁观者的人民终归会有所获益。 事实上，今年的晚会上并没有什么大料爆出，但依然揭了诸如移动这样的大公司的短，也披露了网络隐私、网络犯罪等危害人民日常生活甚至是财产的事情；依然算是给了我们一个交待。不过月夜希望，揭出的黑幕问题能被早日解决，还我们一个朗朗乾坤。

4.“香水女孩”事件

今年3月4日至3月13日，北外一个名为“香奈儿”的女生在其QQ空间中发文言辞激烈地批判中国的外语教育，将矛头直指教育部；并提出诸如“我推荐赵本山当教育部长”的言论。此论一出，立刻在网友中引起共鸣。随后，3月16日凌晨，“香奈儿”又在其博客中发表题为《妈妈，我被北外强制退学了》的文章，披露因自己发表指责教育部的文章，被老师找去谈话，要求删除博文，并被北外强制退学。这篇文章更是极大地调动起了网友们愤怒的情绪，纷纷指责教育部和北外的恶劣行径，要求人肉搜索相关老师。最后，香水女孩和北外达成协议，自己在博客中封笔，重返校园继续读书。 纵观整个事件朴树迷离，有很多网友怀疑此事是彻头彻尾的炒作。不过月夜只是记实，对此事不发表任何看法。

5.Windows 7多个版本泄漏

进入3月，承载着微软公司希望的最新一代操作系统——Windows 7的多个Build版本被泄漏，在互联网上掀起了Win7热潮。 随着Windows 7系统Build版本种子的放出，众多网友先后下载试用了这款操作系统。在试用之后，业界反响不一，捧之者有之，贬低者也不少，不过这依然挡不住Windows 7操作系统的流行，以及随之而来的Win7热潮的涌动。目前，有关Windows 7的新闻在互联网上仍然热度不减。

在前文中，月夜为朋友们送上了第一期的WordPress新手问答，从反馈来看，效果还算不错。因此，本文我们来继续这一专题，希望能为朋友们解答更多的具体问题。

需要注意的是，在前面几期的文章中，月夜主要会围绕月夜自己的博客，为朋友们讲述自己的使用经验，介绍自己对WordPress主题订制等方面的技巧和问题。更多的问答，还请继续关注月夜后来的文章。

四、问：我的博客备案通过了，但不知道怎么在博客中添加备案信息，你能帮帮我吗？

答：使用国内的空间必须面对备案问题，通过了备案的博客或者网站都需要在页面底部添加超链接形式的备案信息。

具体的添加方法如下：

1.通常的主题：比如月夜使用的bbtheme，很受欢迎的pixeled，可能还包括你使用的主题，直接在WordPress主题文件夹下找到footer.php文件，使用Notepad2等非记事本编辑器打开文件，可以看到如下的代码：

Copyright © 2005 - <?php the_time(‘Y’); ?> <?php bloginfo(‘name’); ?> All rights reserved | Powered by Wordpress

或者如：

Powered by WordPress and pixeled by samk.

上面这些都是博客的版权信息，我们通常要将备案信息添加到这些代码里面。如第一个所示，可以将代码

辽ICP备09002157号

添加到All rights reserved前面即可；如第二个，则将如上代码添加到powered by前面即可。当然，添加如上代码时，需要将月夜的备案信息修改为你自己的备案号。

2.特殊主题：尽管80%以上的主题都可以用上述方法解决，但毕竟还是有特例。比如很多人在使用的iNove主题，在这里月夜也以其为例。

iNove主题添加备案信息时，你首先可以参阅月夜的文章iNove主题本地化的几点经验(二)中的9.修改底部的一些信息，在那里，月夜详细讲述了如何修改iNove主题的底部信息。添加备案信息时，只需将那里我们修改好的“备案中”三个字替换为你的备案号即可。

Posted on 2009-04-11 | In 网站相关

月夜在前面的文章中已经不止一次试图通过自己的语言为更多朋友更好的用上WordPress而努力，先后推出了一系列面向WordPress新手的文章。但根据反馈情况来看，效果并不是特别明显。综合考虑了一番之后，月夜认为，可能是因为我们文章中的描述太过宽泛，针对性不够强，使得很多在使用WordPress中出现问题的朋友不能及时解决问题。因此，从本文开始，月夜将自己在使用WordPress中的经验，以及回答很多朋友提出的问题时积累的WordPress问题，综合起来，推出这样一系列文章，希望能对朋友们使用WordPress有所帮助。

一、问：我在安装WordPress时，为什么会出现这样的错误提示“Error establishing a database connection”呢？该怎么才能解决呢？

答：可能有两种情况：

1.WordPress使用的MySQL数据库没有创建好；

2.没有正确配置WordPress程序文件夹中的wp-config.php文件；

相应的解决方案也分两种情况：

1.在你的空间里面创建WordPress使用的MySQL数据库；当然，创建好之后还需要按照2中配置wp-config.php；

2.正确配置wp-config.php文件，如下：

1).将WordPress安装文件夹下的wp-config-sample.php修改为wp-config.php文件；

2).将其中如下的部分：

// ** MySQL设置 - 具体信息来自您正在使用的主机 * //
/\* WordPress数据库的名称 */
define(‘DB_NAME’, ‘putyourdbnamehere’);

/** MySQL数据库用户名 */
define(‘DB_USER’, ‘usernamehere’);

/** MySQL数据库密码 */
define(‘DB_PASSWORD’, ‘yourpasswordhere’);

/** MySQL主机名 */
define(‘DB_HOST’, ‘localhost’);

根据你的主机空间里的MySQL数据库情况具体修改，修改方法可参阅网站建设全过程详细讲解(四)文中的3.创建用于网站建设的MySQL数据库。

二、问：我在安装WordPress时，安装页面出现了这样的提示：Warning: Cannot modify header information，到底是怎么回事儿啊？

答：出现这样的提示有两种可能：

1.使用记事本修改了wp-config.php文件；

2.上传WordPress程序时，没有使用二进制方式上传。

解决方法对应也有两种办法：

1.使用notepad2、ultraedit等编辑器而不要使用记事本来编辑wp-config.php以及所有其他的WordPress程序或者WordPress主题文件；

2.设置FTP客户端上传方式为二进制上传；配置方法可参阅网站建设全过程详细讲解(四)中2.配置FTP工具部分。

与此相似，如果安装或使用WordPress中出现了如“Fatal error: Call to undefined function”等的信息提示，通常也是因为没有使用二进制方式上传。此时，只需设置好FTP客户端，重新上传覆盖原来的文件即可。

三、问：我修改了使用的WordPress主题的某些信息之后，为什么出现了如下图所示的乱码？

答：出现这样的情况有两种可能：

1.使用记事本修改了主题文件；

用好WordPress不可不知的函数（二）

Posted on 2009-04-08 | In 网站相关

在前面的文章用好WordPress不可不知的函数（一）中，月夜已经为你讲述了WordPress的几个非常有用的函数，比如bloginfo()用以输出显示博客信息；wp_title()用以显示页面标题；以及如wp_list_categories()等几个WordPress分类处理的函数；等等。本文，月夜接着前面的文章继续为你介绍WordPress函数；今天，我们主要来介绍一些WordPress的条件函数，用以判断是否满足了某些条件，比如，当前是否在首页；当前是否是某位作者，等等。 8.is_home()

is_home()用以判断当前显示的博客页面是否是博客首页，返回的是一个Bool值。如果是在首页，则返回TRUE；否则返回FALSE。 该函数常用来控制博客侧边栏的显示方式，经常使用如下代码段： <?php if ( is_home() ) { //此为在博客首页应该显示的内容 } else { //此为非博客首页应该显示的内容 } ?>

9.is_archive()

is_archive()用以判断当前显示的内容是否是博客存档页面，比如按日期的存档，或者按分类的存档，等等；其和is_home()函数一样，返回一个Bool值。

10.is_page()

is_page()函数判断当前显示的内容是否是博客的独立页面（page），如“月夜私语”、“关于月夜”等页面；它也返回一个Bool值。 我们可以在模板中通过该函数判断当前是否是一个独立页面，从而决定是否为当前显示的文章显示发布时间等等。

11.is_paged()

该函数用以判断当前文章是否因为内容过多而分页显示；需要注意的是，如果你在写文章时手动添加了标签，来强制分页的话，该函数并不会因此而返回TRUE。

12.is_page_template()

is_page_template()函数需要跟一个参数，通常以如下方式使用： is_page_template(‘guestbook.php’); 藉此判断当前显示的独立页面（page）是否使用了参数所示的模板guestbook.php；如果不跟参数，函数返回当前独立页面是否使用了模板。

13.is_single()

is_single()用以判断当前显示的页面内容是否是一篇单独的文章。其后面可以跟三种参数，一种是文章ID；一种是文章题目（title）；一种是文章名称（slug，文章题目的一种简短说明形式）；或者可以将三种参数组合使用，藉此来判断当前页面内容是否是具体的某篇文章。 一个简单的例子如下，我们可以通过如下几种方式判断当前显示的内容是否是本文： is_single(‘808’); is_single(‘用好WordPress不可不知的函数（二）’); is_single(‘functions-must-known-using-wordpress-second’); is_single(‘808’,’用好WordPress不可不知的函数（二）’,’functions-must-known-using-wordpress-second’); 在这里，月夜需要对上述最后一种方式做些说明：该函数后跟三个参数时，有优先级，如果第一个参数符合条件，则返回TRUE；否则，则返回FALSE；貌似后面的参数并没有什么意义。

14.is_category()

该函数用以判断当前显示的页面内容是否是一个分类页面，如网站相关；其中无需参数。函数返回一个Bool值。

15.is_tag()

is_tag()用以判断当前显示的页面是否是一个标签页面，比如WordPress；其后也不需要跟参数。该函数同样返回一个Bool值。

16.is_date()

此函数用以判断当前显示的内容是否为按时间归档的页面，比如2009年四月，或者2009年4月8日，等等。

17.is_day()、is_month()、is_year()

这些函数用以判断当前显示的内容是否为按天、按月、按年份归档的页面。它们和is_date()类似，只不过将归档时间更具体化而已。

18.is_author()

该函数用以判断当前显示的内容是否为以作者名归档的页面，比如月夜博客的admin作者页面。

19.is_admin()

is_admin()函数用以判断当前是否在控制面板页面，或者管理员面板页面。
